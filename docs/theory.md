# 自己乖離の測定理論
## ─ 本音・建前・本性の三層構造と層間距離の形式化 ─

**ver. 1.0 / 2026-04-28**

---

## Abstract

本稿は、人間の自己を「本性（L0）」「本音（L1）」「建前（L2）」の三層に分割し、各層の間に生じる乖離を連続量として測定するための理論的枠組みを提示する。既存の心理測定は各層を独立に測定するにとどまり、層間の「距離そのもの」を主指標として扱う研究は存在しない。本稿では以下の四点を理論的貢献として提示する。

1. **三層自己モデルの形式化**：各層をベクトル空間上の確率分布として定義し、層間距離（ΔA・ΔB・ΔC）をKLダイバージェンスの変形として計算可能にする。
2. **適応U字モデルの数理化**：ΔBと社会適応の関係が線形でなくU字型であることを損失関数として表現し、個人最適域を導出する。
3. **二欄プローブ法（Dual-Column Probe Method）の測定論的基礎付け**：内省の限界・機械捏造問題・観測者効果を同時に制御する測定設計の原理を明示する。
4. **関係性依存の本音多様体（Relational Honnē Manifold）モデル**：単一の本音ベクトルを仮定せず、関係性×場面×時間の三軸上に本音を多次元的に分布させる。

これらの統合により、アプリケーションが実装すべき計算的・UX的設計指針を導く。

## 1. はじめに：測定されてこなかったもの

### 1.1 問いの起源

心理学は一世紀以上にわたって「自己」を研究してきた。しかし、その研究の大半は「自己とは何か」を問うものであり、「自己の表層と深層がどれだけ離れているか」を日常的・連続的に測定しようとするものではなかった。

本音と建前の乖離は、日本社会において「文化的に管理されたギャップ」として機能してきた（土居, 1971; 中根, 1967）。それは単純な欺瞞ではなく、関係維持と集団秩序のための洗練された社会技術である。問題は、この乖離が当事者にとって**可視化されない**ことにある。人は自分がどれだけ建前に支配されているかを知らない。アプリが測ろうとするのはその「知らなさ」そのものである。

### 1.2 本稿の射程

本稿は心理学・社会学・計算科学の三領域にまたがる。純粋な理論論文としての完結性よりも、実装可能なアプリケーションへの橋渡しを優先し、以下の問いに答える。

- 「本性」「本音」「建前」をどう形式化すれば、その間の距離を計算できるか。
- 乖離の大きさは何に照らして「大きい」と言えるのか（健全閾値問題）。
- 内省データ（自己申告）の限界を、どう設計で補正するか。
- 測定行為そのものがギャップを変えてしまう逆説に、どう応答するか。

## 2. 概念的基礎

### 2.1 三層自己モデル

自己を以下の三層で定義する（表1）。この分割は後述する形式化のための作業仮説であり、実際の自己が離散的に三層に分かれると主張するものではない。

| 層 | 名称 | 定義 | 自覚度 |
|----|------|------|--------|
| L0 | 本性（Core） | 無意識的傾向。認知バイアス、情動の自動反応、防衛機制、価値観の根。本人すら明瞭に自覚していない。 | ほぼゼロ |
| L1 | 本音（Honnē） | 自覚されているが選択的に開示される欲求・感情・意見。「言わないだけで分かっている層」。 | 高い |
| L2 | 建前（Tatemae） | 場面・関係性に応じて社会的に呈示される自己。役割、スクリプト、SNS上のキャラクターを含む。 | 完全に意識化 |

**ユング対応**：L2 ≈ ペルソナ、L0 ≈ シャドウ（Jung, 1928）
**ウィニコット対応**：L2 ≈ False Self、L0 ≈ True Self（Winnicott, 1960）
**ロジャース対応**：L2 = 表現された自己、L1 = 意識された自己、L0 = 経験している自己（Rogers, 1961）

### 2.2 三つのギャップ

三層間に三種類のギャップを定義する。

```
ΔA = d(L0, L1)   ─ 自己理解のギャップ（本性と本音の乖離）
ΔB = d(L1, L2)   ─ 自己呈示のギャップ（本音と建前の乖離）
ΔC = d(L0, L2)   ─ 総合的乖離（本性と建前の乖離）
```

ΔCはΔA+ΔBに等しくない。距離関数の選択と、L0・L1・L2の関係性（独立か非独立か）によって異なる値をとる。この点は §3 で形式化する。

### 2.3 ΔBのU字モデル：適応最適域の存在

ΔBと社会適応の関係は直線的でない。図1に示すU字型のコスト関数を仮定する。

```
        コスト（不適合）
           │
    ┌──────┤
    │      │      ┐
    │      │      │ ΔB=0 の不適合：
    │      │      │ 本音をそのまま出す。率直だが
    │      │      │ 摩擦を生む（空気が読めない型）
    │      │
    │      │           ┐
    │      │           │ ΔB過大の不適合：
    │      │           │ 本音を全く出さない。
    │      │           │ 過剰適応・自己疎外・空虚感
    │      │           │
    └──────┴───────────── ΔB
          最適域
         （適度な
          ギャップ）
```

**命題1**：ΔBに関するコスト関数C(ΔB)は凸（U字型）であり、最小値をとる最適域 [ΔB_lo, ΔB_hi] が存在する。この最適域は文化・関係性・場面によって異なり、普遍的な基準値は存在しない。

この命題は「ΔBを減らすことが善」という直観を否定する。アプリが提示するのはスコアではなく「あなたは今どちら側に寄っているか」という方向情報である。

### 2.4 ΔA × ΔB 適応マトリクス

ΔAとΔBを直交軸として組み合わせると、自己適応の四象限が生まれる。

```
             ΔB 小                  ΔB 大
         （本音優位）           （建前優位）
         ┌──────────────────────────────────┐
ΔA 小   │ 素直だが摩擦多い     │ 自覚的な演技     │
（自己  │ 率直型               │ 戦略型適応       │
 理解  ├──────────────────────────────────┤
 高）  │                      │                  │
ΔA 大  │ 無自覚な攻撃性       │ 最も危険         │
（自己 │ 衝動的不適合         │ 空虚・離人       │
 盲目）│                      │ 燃え尽き         │
        └──────────────────────────────────┘
```

右下象限（ΔA大・ΔB大）が最も臨床的リスクが高い。日本社会では文化的圧力がΔBを大きく保つ方向に作用するため、ΔAの小さい戦略型適応（右上）が健全な均衡点と考えられる。

## 3. 形式的定義

### 3.1 自己状態の表現

各層を**感情-認知ベクトル空間** V ⊆ ℝⁿ の上の確率分布として表現する。

```
L_k ~ p_k(v | c)
```

ここで：
- v ∈ V：感情・認知・意図の特徴ベクトル
- c ∈ C：コンテキスト（誰と、どこで、何時）
- k ∈ {0, 1, 2}

**重要な制約**：
- L2 は観測可能（発言・行動ログ）
- L1 は自己申告で近似可能（二欄プローブ法）
- L0 は直接観測不能。行動パターン・反応時間・生体信号から間接推定

### 3.2 層間距離の定義

距離関数 d(L_i, L_j) を以下で定義する。

#### ΔB の計算（本音-建前距離）

L1とL2が観測可能である場合、対称KLダイバージェンスを用いる。

```
ΔB(c) = D_sym(p₁(· | c) ‖ p₂(· | c))
       = D_KL(p₁ ‖ p₂) + D_KL(p₂ ‖ p₁)
```

二欄プローブ法の実装では、テキスト特徴から分布を推定する。MVP段階では文字数・感情語頻度・強度スコアの加重和で近似する。

#### ΔA の計算（本性-本音距離）

L0は直接観測できない。したがって：

```
ΔA = d(L0_hat, L1)
```

L0_hat は行動ログ・反応時間・選択パターンから推定される潜在変数である。詳細は §5 で述べる。

#### ΔC の計算（総合乖離）

三角不等式より：
```
ΔC ≤ ΔA + ΔB
```

等号は L0, L1, L2 が同一次元で同一方向に整列している場合のみ成立する。実際には L0 と L2 の乖離は、L1 を迂回する経路よりも直線距離の方が短い場合がある（すなわち ΔC < ΔA + ΔB）。これはL1が本音の部分的・歪んだ反映にすぎない場合に起きる。

### 3.3 コンテキスト依存性

距離はコンテキスト c に依存して変化する。

```
ΔB: C → ℝ₊
c ↦ ΔB(c)
```

コンテキストの構成要素は少なくとも以下を含む。

```
c = (r, s, t)
```

- r ∈ R：関係性（上司, 親, 恋人, …）
- s ∈ S：場面（会議, 食事, SNS, …）
- t ∈ T：時刻

アプリが目指すのはこの関数 ΔB(c) のマッピングである。単一の ΔB スコアを個人に付与するのではなく、コンテキスト空間上に ΔB を分布として描き出す。

## 4. 測定論

### 4.1 内省の限界問題とその設計的解

Nisbett & Wilson（1977）は、人が自己の意思決定理由を後付けで作話することを示した。これは内省データの信頼性に根本的疑問を投げかける。しかし、この批判は「内省が無意味」を主張するのではなく、「内省だけでは不十分」を主張する。

本アプリの対応戦略：

| 問題 | 対応 |
|------|------|
| 作話（post-hoc rationalization） | 反応時間の計測（速い反応は無意識に近い） |
| 社会的望ましさバイアス | 二欄法の本音欄を意図的に「評価されない文脈」に設定 |
| 内省の語彙不足 | 既製の感情語リストを補助的に提示（命名の手がかり） |
| 観測者効果 | 後述 §4.3 |

### 4.2 二欄プローブ法（Dual-Column Probe Method）の測定論的位置

二欄プローブ法は以下の三性質を同時に満たす、これまでにない測定設計である。

**性質1：測定と介入の同時性**
二欄を並列に書く行為自体が、未言語化の本音の言語化（ジャーナリング療法と同型）として機能する。測定が同時に気づきを生む。これは「測定の副作用」ではなく設計の核心である。

**性質2：機械捏造問題の無効化**
「AIが本音を当てる」設計では、提示された「本音」が本人の本音なのか、機械が生成した暗示なのか区別できない（Rosenhan, 1973 の精神診断問題に類似）。二欄法では発見の主体を常にユーザーに保つことで、この問題を構造的に排除する。

**性質3：ΔBの直接測定**
既存の心理測定は L1 か L2 のいずれか一側面を測るが、二欄法は L1（本音欄）と L2（建前欄）を同一測定文脈で並列取得する。これにより ΔB の直接推定が可能になる。

**形式化**：
状況カード s を提示したとき：

```
Probe(s) = (h(s), t(s), i(s))
```

- h(s) ∈ T_text：本音の自由記述
- t(s) ∈ T_text：建前の自由記述
- i(s) ∈ [0, 10]：感情強度

ΔB_probe(s) = f(h(s), t(s), i(s))

f は特徴抽出関数であり、MVP段階では以下を採用する。

```
f = w₁ · |len(h) - len(t)| / max_len
  + w₂ · sentiment_diff(h, t)
  + w₃ · i(s) / 10
```

重み w₁, w₂, w₃ はユーザーの蓄積データからベイズ的に更新する。

### 4.3 観測者効果（自己観察のパラドックス）

本アプリ最大の哲学的困難は、ギャップを観察する行為がギャップを変えてしまうことである。これはハイゼンベルク的制約として原理的に回避不能である。

**三つの応答戦略**：

*戦略A：受動センシング優先*
ユーザーの能動的入力を最小化し、行動ログ・通知パターン・アプリ使用時間などのパッシブデータを主としてΔBを推定する。観測者効果を最小化できるが、精度とプライバシーの問題がある。

*戦略B：観測後自己の正常化*
「観察されたあとの自己」を新しいベースラインとして取り込む。観測者効果を制御するのではなく、変化のベクトルを積極的にトラッキングする。「今日の観察によってどこが変わったか」を追う設計。

*戦略C：意図的な再観察サイクル*
同じ状況カードを週・月単位で再提示し、記述の変化を観測する。観測者効果を縦断データとして記録する。変化量が「気づきの量」の代理指標になる。

本アプリは戦略Bと戦略Cを組み合わせる。ΔBの絶対値よりも「ΔBの変化速度」を主要出力とすることで、観測者効果をシグナルに変換する。

## 5. 本性（L0）の間接推定

### 5.1 L0の存在論的立場

本稿は「本性は存在する」という実在論的立場をとらず、以下の操作的定義を採用する。

**定義（本性の操作的定義）**：
L0 は、時間・場面・関係性にまたがって統計的に安定した自己応答パターンの集合である。

```
L0 = {v ∈ V : Var_c[p(v | c)] < θ}
```

すなわち、コンテキスト c が変化しても変動しにくい特徴の集合を「本性」と呼ぶ。この定義により、「本性が変わった」という発達的変化と「本性が現れた」という発見的変化を、データの統計的振る舞いとして区別可能になる。

### 5.2 L0推定のための間接指標群

| 指標 | 対応するL0の側面 | 実装難度 |
|------|----------------|---------|
| 反応時間（RT） | 自動的・無意識的な感情反応 | 中（ブラウザ計測可） |
| 語彙連想課題 | 概念と感情の暗黙的連合 | 中 |
| 選択の一貫性 | 価値観の安定性 | 低（行動ログ） |
| カードの選択パターン | 何を「ある」と感じるか | 低（既存機能の延長） |
| 記述の語調変化 | 感情の自動的な漏出 | 中（LLM分析） |

### 5.3 L0推定モデル

蓄積されたエントリーから L0 を推定するベイズフィルタを設計する。

```
p(L0 | D₁:ₙ) ∝ p(D₁:ₙ | L0) · p(L0)
```

- D₁:ₙ：n個のエントリー（本音記述・建前記述・感情強度・関係性タグ）
- p(L0)：事前分布（初期は無情報事前分布）
- p(D | L0)：尤度関数（本音記述がL0から生成されたとする生成モデル）

MVP段階では簡略化し、本音記述から抽出した感情語ベクトルのコンテキスト非依存成分を L0_hat として用いる。具体的には主成分分析の第一主成分（最大分散方向）がL0の代理指標になる。

## 6. 関係性依存の本音多様体

### 6.1 単一本音ベクトルの問題

古典的自己概念研究は、人が統一された「本音」を持つと暗黙に仮定している。しかし経験的には：

- 親には言える本音 ≠ 恋人には言える本音
- 職場での本音 ≠ 友人の前の本音
- どれも「本音」である

この問題を「本音のどれが本物か」と問うのは誤りである。正しい問いは「本音はどの次元で関係性依存しているか」である。

### 6.2 関係性依存の本音多様体（Relational Honnē Manifold）

本音の空間を多様体として定義する。

```
M_H = {h(r, s, t) : r ∈ R, s ∈ S, t ∈ T}
```

- h(r, s, t)：関係性 r、場面 s、時刻 t における本音ベクトル
- M_H は V 内の部分多様体

この多様体の幾何学的性質が、その人の本音の構造を表す。

**キャラクタリゼーション**：

1. **多様体の次元**：本音のバリエーション数。次元が高いほど、関係性ごとに異なる本音を持つ（多面的）。低いほど一貫している（単純）。

2. **多様体の曲率**：隣接した関係性間での本音の変化速度。曲率が高い箇所は「文脈の切り替えコストが高い」を意味する。

3. **多様体の中心**：全関係性を通じた本音の平均。これが L0 の代理推定量として機能しうる。

### 6.3 アプリへの実装

MVP段階での近似：関係性タグ（12カテゴリ）別に本音記述の感情語分布を集計し、カテゴリ間のKLダイバージェンスを隣接行列として関係性マップを描く。

```
RelationalMap[r_i, r_j] = D_sym(p_honne(· | r_i) ‖ p_honne(· | r_j))
```

この行列のヒートマップが「誰の前の自分が誰の前の自分と一番違うか」を可視化する。

## 7. 時間的ダイナミクス

### 7.1 ギャップの時系列モデル

ΔB は静的な量ではなく、時間とともに変動する。

```
ΔB(t) = ΔB_baseline + ε(t) + drift(t)
```

- ΔB_baseline：個人の構造的なベースライン
- ε(t)：短期変動（日次・週次のコンテキスト変化）
- drift(t)：長期的ドリフト（成長・加齢・関係性の変化）

### 7.2 臨界点の検出

ΔB の時系列において以下の臨界点を検出する。

**急増イベント**：ΔB が短期間で急上昇した場合、強いストレス・抑圧・関係性の変化を示唆する。

**急減イベント**：ΔB が突然低下した場合、安全な関係性の発見、または解離的な感情麻痺を示唆する（符号が重要）。

**ベースライン漂流**：数週間以上にわたる単調増加は、過剰適応への構造的移行を示唆する。

### 7.3 「気づきの波」モデル

二欄プローブ法を繰り返すことで、本人の本音への言語アクセスが向上する。これを「気づきの波」として定式化する。

```
clarity(t) = 1 - exp(-λt) · cos(ωt)
```

- λ：気づきの蓄積速度（個人差）
- ω：内省サイクルの周波数

初期はcos項による揺れが大きい（日によって見える・見えない）が、時間とともに収束する。この収束速度λがアプリのコアな価値指標になる。

## 8. 倫理的枠組み

### 8.1 自律性の保護

ギャップの測定は、本人が「知りたい」と意図したものだけを開示する。強制的な気づきは心理的危機を引き起こしうる（防衛機制の突然の崩壊）。

**設計原則1**：ΔAが急に大きく推定された場合、直接提示せず「最近、予想外の反応をすることはありますか？」のような間接的プロンプトに留める。

**設計原則2**：データの削除権を常に保証する。「なかったことにする」は自己理解の後退ではなく、ペースの調整である。

### 8.2 第三者データの問題

「上司Aの前での自分の本音」を記録することは、上司Aに関するデータを副次的に生成する。本人のデータでありながら、他者の行動特性情報を含む。

**設計原則3**：関係性タグは固有名詞でなくロール名（「上司」「恋人」）のみで記録する。特定個人の識別情報をシステムが処理しない。

### 8.3 臨床境界の定義

以下の状態を検出した場合、アプリは専門家連携を促す。

```
Alert condition:
  ΔB > percentile_95(ΔB_population) AND
  ΔA > percentile_80(ΔA_proxy) AND
  duration > 4 weeks
```

この条件は「過剰適応 × 自己盲目 × 長期継続」の三重重なりを捉える。アプリは診断を下さず「最近、誰かに話せていますか？」と問いかけるのみにとどめる。

### 8.4 データの権力構造

ΔBデータは、権力者が部下の「建前の厚さ」を知るために悪用されうる。組織向けに集計データを提供する場合は個人識別が不可能な形式に限定し、管理者は部下の個別データへアクセスできないことを技術的に保証する。

## 9. 文化的補正

### 9.1 日本社会のΔBバイアス

日本社会では ΔB を大きく保つことが社会規範として内面化されている。したがって、日本人ユーザーの ΔB 分布は欧米のそれと比較して右側に偏ることが予想される。

この偏りは「異常」ではなく「文化的正常値」である。アプリは普遍的規範値との比較ではなく、以下の基準を採用する。

1. **個人内ベースラインとの比較**：自分の通常 ΔB からの偏差
2. **同一関係性カテゴリ内の変動**：上司との ΔB が先週より高くなっているか
3. **自己申告の主観的苦痛度**：ΔB の絶対値ではなく、それが苦しいかどうか

### 9.2 ΔBの文化的較正モデル

```
ΔB_normalized(i, c) = (ΔB(i, c) - μ_cultural(c)) / σ_cultural(c)
```

- ΔB(i, c)：個人 i のコンテキスト c における ΔB
- μ_cultural(c)：コンテキスト c における文化的期待値
- σ_cultural(c)：文化内での標準偏差

MVP段階では μ_cultural と σ_cultural を固定値（文献ベースの推定値）として設定し、データ蓄積後に更新する。

## 10. 計算設計の指針

本稿の理論から導かれる実装指針を列挙する。

### 10.1 何を計算するか

| 指標 | 計算対象 | 優先度 |
|------|---------|--------|
| ΔB(r) | 関係性別の本音-建前距離 | ★★★（MVP） |
| ΔB(t) | 時系列での変化 | ★★★（MVP） |
| RelationalMap | 関係性間の本音類似度行列 | ★★（v2） |
| clarity(t) | 気づきの蓄積量 | ★★（v2） |
| L0_hat | 本性の間接推定 | ★（v3） |
| ΔA | 本性-本音距離 | ★（v3） |

### 10.2 計算の誠実さ原則

- 計算根拠をユーザーが反証可能な形で示す
- 推定値には常に信頼区間を付与する
- 「データが少なすぎる」を明示する（エントリー数 < 5 なら数値を非表示）

### 10.3 状況カードの設計原則

状況カードはΔBのサンプリングを均一にするための刺激である。設計時の注意：

- 各関係性カテゴリに均等に配布する
- 感情価が正・負・中性の三種を含む
- 具体的すぎない（個人が同定されるレベルは避ける）
- 抽象的すぎない（投影対象が定まらなくなる）

## 11. 未解明領域と今後の問い

### 11.1 本音の粒度問題

「本音」は命題（「上司Aが嫌いだ」）なのか、情動（「何か不快な感覚」）なのか、それとも身体感覚（「胃が重い」）なのか。この粒度の問いに答えが出るまで、L1の定義は曖昧さを含む。

### 11.2 集合的本音の問題

カップル・チーム・家族においては「私たちの本音」が存在するのか。集合的主体における三層モデルの拡張は未着手である。

### 11.3 本音の可塑性と真正性

「以前より本音が変わった」は「本性が変わった」のか「本音のアクセスが深まった」のか。この問いに答えるためには、L0とL1を独立に計測する縦断研究が必要であり、現時点では原理的に判別不能である。

### 11.4 ΔB最適域の個人差

U字モデルの最低点（最適ΔB）は個人によって異なる。高感受性者・愛着スタイル（Ainsworth, 1978）・MBTI的傾向との相関が予想されるが、実証データは存在しない。本アプリのデータ蓄積から初めて検討可能になる問いである。

## 12. 結論

本稿は「本音と建前のギャップを連続量として測定する」という問いに対して、以下を提示した。

1. 三層（L0/L1/L2）の形式的定義と、層間距離（ΔA/ΔB/ΔC）の計算可能な表現。
2. ΔBのU字コストモデルによる「適応最適域」の定式化。これにより「ΔBを減らすことが善」という誤った設計指針を理論的に棄却できる。
3. 二欄プローブ法の測定論的基礎付け。内省の限界・機械捏造問題・観測者効果への設計的応答。
4. 本音の多様体モデルによる「関係性依存の本音」の表現。単一本音ベクトルの仮定を廃棄し、関係性×場面×時間の三軸上に本音を分布として扱う。
5. 文化的補正の必要性と、個人内ベースライン比較の優位性。

これらの理論は、アプリが何を計算し何を計算しないかを規定する設計憲法として機能する。完成形を最初から定義せず、未解明と並走しながら、データの蓄積とともに理論を更新していく姿勢が、このプロジェクトの知的誠実さを保証する。
